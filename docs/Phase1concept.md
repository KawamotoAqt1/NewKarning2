# 🚀 **Phase1 コンセプト：

“重み学習”による **ロジック精度の底上げフェーズ**

---

## 🎯 **Phase1 の位置づけ**

### **Phase0 → Phase1 で何が変わるのか？**

| 項目      | Phase0（今）            | Phase1（次）                     |
| ------- | -------------------- | ----------------------------- |
| カーニング手法 | すべてルールベース（手作業で決めた係数） | **ルールに対して“効き具合”を自動学習**        |
| 精度      | ある程度改善 / まだムラ        | **ムラが減り、デザイナーの感覚に近づく**        |
| 主目的     | PoC・方向性の確認           | **精度の底上げ & 商用品質への橋渡し**        |

| 学習データ   | なし                   | **AI（Illustrator）デザインから自動抽出** |

Phase1 は、
**「ルールの限界」を“学習で補正する”ステップ**です。

---

# 1️⃣ **Phase1 の目的**

### **目的は1つだけ：**

> **Phase0 のロジックに “重み” を学習で付与し、
> デザイナーの仕上げに近づける。**

ここでは深層学習は不要。
必要なのは **軽い線形回帰による係数最適化**だけで十分。

---

# 2️⃣ **Phase1 で新しくやること**

## ■ 2-1. 「デザイナーが実際に作った AIファイル」から文字間隔を自動抽出

対象フォントは代表3〜5個でOK。

**1フォントあたり10〜20デザイン × 最大5フォント**
→ **30〜60デザイン**で Phase1 の精度は達成可能。

抽出情報：

* 各文字ペアの実際の間隔 `gap_actual`
* Phase0 のロジックが出す `gap_predicted`
* 文字特徴量（Phase0と同じ：charType, density, aspectRatio など）
* ペアカテゴリ（VERTICAL_VERTICAL等）

---

## ■ 2-2. **gap のズレを最小化する “重みw” を学習**

Phase0 の公式：

```
gap_predicted =
    baseTracking
  + pairAdjust
  + densityAdjust
  + styleAdjust
  + globalTracking
```

Phase1 の公式：

```
gap_final =
    baseTracking
  + w_pair    * pairAdjust
  + w_density * densityAdjust
  + w_style   * styleAdjust
  + globalTracking
```

### ✔ **学習するものは “重み” だけ**

（ロジックそのものは変えない）

* `w_pair`（ペア補正の効き具合）
* `w_density`（黒密度補正の効き具合）
* `w_style`（テンプレ世界観補正の効き具合）

全部合わせて **10〜20個程度の軽いパラメータ**。

---

## ■ 2-3. 学習方法は “線形回帰” でよい

モデルは単純で良い：

```
delta = gap_actual - gap_predicted
```

これを最小にするように `w_pair, w_density, ...` を求める。

つまり：

* 機械学習のコストは極小
* 学習データは少数でOK
* 過学習のリスクも低い
* GPU不要、ブラウザで学習させることも可能

# 3️⃣ **Phase1 の成果物 (アウトプット)**

Phase1 が終わると以下が揃う：

### ✔ 1. **重み付きロジックの完成**

* ペア別の重み
* 黒密度別の重み
* テンプレ別の重み

### ✔ 2. **Phase0より明らかにムラが減る**

* 特に「縦線×縦線」「丸×丸」などでの精度改善が実感できる
* デザイナー修正量が大幅に減る

### ✔ 3. **このフェーズで商用に耐える領域が見えてくる**

* よく出る名字 × よく使うフォントはかなり安定する
* 売れ筋テンプレでは実際に商品化できる品質に近づく

# 4️⃣ **Phase1 のKPI（成功基準）**

| KPI                     | 基準値        |
| ----------------------- | ---------- |
| デザイナーが「微調整だけでOK」になる割合   | **70〜80%** |
| Phase0との比較で “見た目の改善が明確” | YES        |
| ロジックの破綻（極端な空き/詰まり）がない   | 0件         |

---

# 5️⃣ **Phase1 に入れるとやりすぎになるもの（やらない）**

### ❌ 3文字以上の「全体最適」

→ Phase2 で実施

### ❌ フォント100種類の学習

→ Phase1は代表フォントのみで十分

### ❌ 深層学習によるエンドツーエンド学習

→ 計算コストもデータ量も見合わない

### ❌ 例外パターンの大量登録

→ Phase3で運用フェーズとして吸収

Phase1はあくまで **“ロジックを人間に近づけるための微調整”**。

---

# 6️⃣ **Phase1 の工数感・期間**

### 期間：

**約1.5〜2.5週間**

### 内訳：

1. AIファイルからgap抽出ツール作成：3日

# 7️⃣ **Phase1 完了後の未来（Phase2 の方向性）**

Phase1 の良さは、
**Phase2 全体最適（重心補正、標準偏差補正など）が入る前に
ロジックの“素地”を整えること。**

Phase1 があるからこそ Phase2 でスムーズに精度が跳ね上がる。
